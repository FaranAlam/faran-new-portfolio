\section{Design Tokens Foundation}

\begin{infobox}{What are Design Tokens?}
Design tokens are the \textbf{named variables} that store all visual decisions in your design system -- colors, font sizes, spacing, shadows, border radius. They create a \textbf{single source of truth} shared between design (Figma) and code (CSS/Tailwind). Change a token = change the whole system.
\end{infobox}

\begin{tipbox}
\textbf{60-30-10 Color Rule:} 60\% neutral/background, 30\% secondary color, 10\% primary/accent. This ratio creates balance automatically. Never use your primary color more than 10\% of the page area.
\end{tipbox}

\section{Component States}

Every interactive component needs \textbf{5 states} designed:

\begin{center}
\begin{tabularx}{\textwidth}{|>{\bfseries\color{FaranViolet}}l|X|X|}
\hline
\rowcolor{FaranDark}
\color{white}\textbf{State} & \color{white}\textbf{When} & \color{white}\textbf{Visual Change} \\
\hline
Default & Normal idle state & Base appearance \\
\hline
\rowcolor{SectionBg}
Hover & Mouse over element & Slightly darker bg, shadow \\
\hline
Active/Pressed & Being clicked & Even darker, slight scale down \\
\hline
\rowcolor{SectionBg}
Focus & Keyboard focused & Visible focus ring (accessibility!) \\
\hline
Disabled & Cannot be interacted with & 50\% opacity, no-drop cursor \\
\hline
\rowcolor{SectionBg}
Loading & Action in progress & Spinner, skeleton, reduced opacity \\
\hline
Error & Invalid input & Red border, error message below \\
\hline
\rowcolor{SectionBg}
Success & Valid / completed & Green indicator, checkmark \\
\hline
\end{tabularx}
\end{center}

\section{Accessibility (A11y) Standards}

\begin{infobox}{Why Accessibility is Non-Negotiable}
In 2026, accessibility is legally required in many countries. Beyond legal compliance, accessible design means \textbf{better UX for all users} -- clear focus states help keyboard users, high contrast helps outdoor mobile users, semantic HTML helps screen readers AND SEO.
\end{infobox}

\subsection{Dark Mode Implementation}

\begin{tokenbox}{Tailwind Dark Mode Setup (CSS Variables)}
\textbf{tailwind.config.ts:} \texttt{darkMode: 'class'}\\[0.3cm]
\textbf{Usage in components:}\\
\texttt{className="bg-white dark:bg-gray-900 text-gray-900 dark:text-white"}\\[0.3cm]
\textbf{Toggle dark mode in layout:}\\
\texttt{document.documentElement.classList.toggle('dark')}\\[0.3cm]
\textbf{Remember to also handle:} borders, shadows, input backgrounds, placeholder text, card backgrounds, scrollbar colors, and image brightness (\texttt{dark:brightness-90}).
\end{tokenbox}

\vspace{0.4cm}
\begin{tipbox}
Test your dark mode by setting \texttt{prefers-color-scheme: dark} in Chrome DevTools (Rendering tab). Also test with OS dark mode. Many developers ship broken dark modes because they only test with the toggle button.
\end{tipbox}
