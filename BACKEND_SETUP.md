## 🛠️ Setup Instructions

### Step 1: Create MongoDB Atlas Account (FREE)

1. **Go to:** https://www.mongodb.com/cloud/atlas
2. **Sign up** with Google/Email
3. **Create a FREE cluster** (M0 tier - 512MB free)
4. **Choose provider:** AWS (recommended)
5. **Choose region:** Closest to you (e.g., Singapore, Mumbai)
6. **Cluster name:** Keep default or use `Portfolio`

### Step 2: Configure Database Access

1. **Go to:** Database Access → Add New Database User
2. **Authentication:** Username and Password
   - Username: `portfolioUser` (or your choice)
   - Password: Generate secure password (save it!)
3. **Database User Privileges:** Read and write to any database
4. **Add User**

### Step 3: Configure Network Access

1. **Go to:** Network Access → Add IP Address
2. **Choose:** Allow Access from Anywhere
3. **IP Address:** `0.0.0.0/0`
4. **(Optional)** Or add your specific IP for more security
5. **Confirm**

### Step 4: Get Connection String

1. **Go to:** Database → Connect
2. **Choose:** Connect your application
3. **Driver:** Node.js, Version 5.5 or later
4. **Copy connection string:**
   ```
   mongodb+srv://<username>:<password>@cluster0.xyz.mongodb.net/
   ```
5. **Replace:**
   - `<username>` with your database username
   - `<password>` with your database password

### Step 5: Setup Local Environment

1. **Create `.env.local` file** in project root:
   ```bash
   cp .env.local.example .env.local
   ```

2. **Edit `.env.local`** and add your MongoDB URI:
   ```env
   MONGODB_URI=mongodb+srv://portfolioUser:YOUR_PASSWORD@cluster0.xyz.mongodb.net/portfolio
   ```

3. **Install dependencies** (if not already done):
   ```bash
   npm install mongodb
   ```

### Step 6: Test Locally

1. **Start dev server:**
   ```bash
   npm run dev
   ```

2. **Open browser:** http://localhost:3000

3. **Go to Contact section** and submit a test message

4. **Check MongoDB Atlas:**
   - Go to Database → Browse Collections
   - You should see `portfolio` database
   - Inside: `contacts` collection with your test message

## 🎯 API Endpoints Documentation

### POST `/api/contact`
Submit a contact form message.

**Request Body:**
```json
{
  "name": "John Doe",
  "email": "john@example.com",
  "subject": "Project Inquiry",
  "message": "I want to build a website..."
}
```

**Success Response (200):**
```json
{
  "success": true,
  "message": "Message sent successfully!",
  "id": "507f1f77bcf86cd799439011"
}
```

**Error Response (400/500):**
```json
{
  "error": "Error message here"
}
```

### GET `/api/contact`
Retrieve all contact messages (for future admin dashboard).

**Success Response (200):**
```json
{
  "success": true,
  "messages": [
    {
      "_id": "507f1f77bcf86cd799439011",
      "name": "John Doe",
      "email": "john@example.com",
      "subject": "Project Inquiry",
      "message": "I want to build...",
      "createdAt": "2026-02-24T10:30:00.000Z",
      "read": false,
      "replied": false
    }
  ]
}
```

## 🚀 Deployment to Vercel

### Step 1: Push to GitHub

```bash
git add .
git commit -m "Added backend with MongoDB integration"
git push origin main
```

### Step 2: Deploy to Vercel

1. **Go to:** https://vercel.com
2. **Import your GitHub repository**
3. **Vercel will auto-detect** Next.js project
4. **Add Environment Variables:**
   - Go to: Project Settings → Environment Variables
   - Add: `MONGODB_URI` = your connection string
5. **Deploy!**

### Step 3: Verify

- Your site will be live at: `your-username.vercel.app`
- Contact form will work on production!
- Messages saved to MongoDB Atlas

## 📊 Database Schema

### Collection: `contacts`

```typescript
{
  _id: ObjectId,              // Auto-generated by MongoDB
  name: string,               // User's name
  email: string,              // User's email
  subject: string,            // Message subject
  message: string,            // Message content
  phone: string | null,       // Optional phone number
  createdAt: Date,            // Timestamp when message was sent
  read: boolean,              // Has admin read this? (false by default)
  replied: boolean            // Has admin replied? (false by default)
}
```

## 🔧 Troubleshooting

### Issue: "Cannot find module 'mongodb'"
**Solution:** Run `npm install mongodb`

### Issue: "MongoServerError: Authentication failed"
**Solution:** 
- Check username/password in `.env.local`
- Verify Database User was created in Atlas
- Password might contain special characters - try URL encoding

### Issue: "MongoServerError: IP not whitelisted"
**Solution:**
- Go to Network Access in MongoDB Atlas
- Add your IP or allow `0.0.0.0/0`

### Issue: "ENOTFOUND cluster0.xyz.mongodb.net"
**Solution:**
- Check your internet connection
- Verify connection string is correct
- Try restarting dev server

### Issue: "Cannot connect to MongoDB in production"
**Solution:**
- Make sure `MONGODB_URI` is added in Vercel Environment Variables
- Redeploy after adding environment variables
